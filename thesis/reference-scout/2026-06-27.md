# Reference Scout — 2026-06-27

## Top References

### 1. Unreal Engine — Root Motion Documentation

- **Type:** official engine docs
- **Link:** https://dev.epicgames.com/documentation/en-us/unreal-engine/root-motion-in-unreal-engine
- **Thesis fit:** Build-first
- **Supports:** Animation / Combat / Implementation
- **Why it matters:**
  - 直接解释 root bone motion 如何驱动 character movement。
  - 适合 animation-driven combat：攻击位移、冲刺攻击、boss lunge attack。
  - Root Motion from Montages Only 对 melee attack montage 很关键。
- **How to use in thesis:** Animation Implementation 章节：解释为什么使用 root motion 而不是纯代码移动。
- **Caveat:** UE 具体实现文档，不是 academic citation，但对做系统非常有用。

### 2. Unreal Engine — Animation Notifies Documentation

- **Type:** official engine docs
- **Link:** https://dev.epicgames.com/documentation/en-us/unreal-engine/animation-notifies-in-unreal-engine
- **Thesis fit:** Build-first
- **Supports:** Animation / Combat / Hit Detection / VFX
- **Why it matters:**
  - Animation Notify 适合触发攻击窗口。
  - Notify State 可以表示 weapon active window：Begin 开启 hit detection，Tick 做 weapon trace，End 关闭 hit detection。
  - 也可以同步 footstep、weapon trail、hit VFX、sound。
- **How to use in thesis:** Combat Implementation 章节：attack timing、hit window、parry window 都可以通过 animation events/notifies 驱动。
- **Caveat:** 需要设计 notify naming convention，例如 `AttackWindow_Start` / `AttackWindow_End`。

### 3. Unreal Engine — Traces Overview

- **Type:** official engine docs
- **Link:** https://dev.epicgames.com/documentation/en-us/unreal-engine/traces-in-unreal-engine---overview
- **Thesis fit:** Build-first
- **Supports:** Combat / Hit Detection / Implementation
- **Why it matters:**
  - melee hit detection 常用 sphere trace / capsule trace / line trace。
  - 适合实现武器从上一帧 socket 到当前帧 socket 的 swept trace。
  - 可以支撑 thesis 中的 animation-driven hit detection。
- **How to use in thesis:** Combat System 章节：weapon trace、hit confirmation、debug visualization。
- **Caveat:** 需要进一步找具体 “weapon tracing with sockets” 的实践资料。

### 4. Unreal / Behavior Tree Direction

- **Type:** engine docs / AI architecture
- **Starting point:** https://en.wikipedia.org/wiki/Behavior_tree_%28artificial_intelligence%2C_robotics_and_control%29
- **Thesis fit:** Build-first + academic support
- **Supports:** Boss AI
- **Why it matters:**
  - Boss AI 可以用 Behavior Tree 或 FSM 驱动：idle / approach / circle / attack / recover / punish / phase transition。
  - Behavior Tree 的优势是 modular、debuggable、可视化。
  - 对 boss attack pattern selection 有帮助。
- **How to use in thesis:** AI Architecture 章节：FSM vs Behavior Tree 对比。
- **Caveat:** Wikipedia 只能当入口；正式引用应使用 Colledanchise & Ögren 或 game AI textbook。

### 5. *Introduction to Behavior Algorithms for Fighting Games* (2020)

- **Type:** academic paper
- **Authors:** Ignacio Gajardo, Felipe Besoain, Nicolas A. Barriga
- **Link:** https://arxiv.org/abs/2007.12586
- **Thesis fit:** Core academic
- **Supports:** AI / Combat
- **Why it matters:**
  - 讨论 fighting game AI 中常见行为算法。
  - 包括 FSM、Behavior Tree、Monte Carlo Tree Search 等。
  - 虽然项目是 third-person melee，不是 2D fighting game，但 “对抗型 AI” 逻辑接近。
- **How to use in thesis:** Related Work / AI chapter，用来支撑选择 state machine 或 behavior tree。
- **Caveat:** 不直接教 UE 实现。

### 6. *Designing Game Feel: A Survey* (2020)

- **Type:** survey paper
- **Authors:** Martin Pichlmair, Mads Johansen
- **Link:** https://arxiv.org/abs/2011.09201
- **Thesis fit:** Core academic
- **Supports:** Game Feel / Responsiveness / Combat
- **Why it matters:**
  - 适合给系统提供理论语言。
  - 核心矛盾是：animation commitment vs input responsiveness。
  - 可用于解释 combat 不只是“能打中”，还要 readable、responsive、juicy feedback。
- **How to use in thesis:** Introduction / Evaluation / Design Goals。
- **Caveat:** 不是实现资料，但适合支撑 thesis 论证。

### 7. *Struggle as Flow: Challenge, Design, and Experience in Soulslike Games* (2026)

- **Type:** academic paper
- **Authors:** Zhehao Sun et al.
- **Link:** https://arxiv.org/abs/2604.15318
- **Thesis fit:** Useful academic support
- **Supports:** Souls-like Design / Boss Fight / Difficulty
- **Why it matters:**
  - 新的 Soulslike 体验研究。
  - 重点是 failure、mastery、rhythm、fairness。
  - 对 boss fight 目标很贴：挑战性必须让玩家觉得“公平、可读、可学习”。
- **How to use in thesis:** Design Motivation / Evaluation Criteria。
- **Caveat:** 偏玩家体验分析，不是 implementation guide。

## 今日优先读 / 做

如果今天要实际推进系统，建议顺序：

1. **UE Root Motion docs** — 搞清楚 attack montage 怎么驱动位移。
2. **UE Animation Notifies docs** — 设计攻击窗口、parry 窗口、VFX/SFX 触发点。
3. **UE Traces docs** — 实现 weapon socket trace / hit detection prototype。

```text
Input
→ Attack Montage
→ Root Motion moves character
→ Notify State opens hit window
→ Weapon trace detects enemy
→ Hit reaction / damage / VFX
```
