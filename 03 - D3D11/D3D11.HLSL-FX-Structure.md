# HLSL FX 文件结构

FX 文件（`.fx`）是 D3D Effects 框架的着色器格式，把 Resources、Shaders、Technique 整合进一个文件。

---

## 整体结构

```hlsl
/*** 1. Resources（资源声明）***/
cbuffer ...          // 常量缓冲
Texture2D ...        // 纹理
SamplerState ...     // 采样规则
RasterizerState ...  // 光栅化状态

/*** 2. Data Structures（数据结构）***/
struct VS_INPUT { ... };   // CPU → VS 的数据
struct VS_OUTPUT { ... };  // VS → PS 的数据

/*** 3. Utility Functions（工具函数）***/
float2 get_corrected_uv(...) { ... }

/*** 4. Vertex Shader ***/
VS_OUTPUT vertex_shader(VS_INPUT IN) { ... }

/*** 5. Pixel Shader ***/
float4 pixel_shader(VS_OUTPUT IN) : SV_Target { ... }

/*** 6. Technique（技术/Pass 定义）***/
technique10 main10
{
    pass p0 { ... }
}
```

---

## 数据流全景

```
CPU
  顶点缓冲（Position + UV + Normal...）
  常量缓冲（WorldViewProjection...）
  纹理（ColorTexture）
         ↓
  Input Assembly（按 Input Layout 切分顶点数据）
         ↓
  Vertex Shader（VS_INPUT → VS_OUTPUT）
    mul(Position, WVP) → 屏幕坐标
    传递 UV 等属性
         ↓
  光栅化（插值 VS_OUTPUT 到每个像素）
         ↓
  Pixel Shader（VS_OUTPUT → float4 颜色）
    Texture.Sample(Sampler, UV) → 颜色
         ↓
  Output Merger（写入 Render Target）
```

---

## 各部分详解

### cbuffer（常量缓冲）

整个 Draw Call 共享的数据，对接靠 `register(bN)` 槽号：

```hlsl
cbuffer CBufferPerObject : register(b0)
{
    float4x4 WorldViewProjection : WORLDVIEWPROJECTION;
}
```

详见 [[D3D11.ConstantBuffer]]

---

### Texture + SamplerState

```hlsl
Texture2D ColorTexture;        // 数据容器（只是 float4 数组）
SamplerState ColorSampler      // 采样规则
{
    Filter   = MIN_MAG_MIP_LINEAR;  // 线性插值，平滑
    AddressU = WRAP;                // UV 超出时循环
    AddressV = WRAP;
};
```

`Texture.Sample()` 返回 float4，语义（颜色/法线/深度）由程序员约定。详见 [[D3D11.Texture]]

---

### VS_INPUT / VS_OUTPUT 结构体

连接 Input Layout 和 Shader 的数据包，靠 **Semantic** 对接：

```hlsl
struct VS_INPUT {
    float4 ObjectPosition : POSITION;    // ← 对应 Input Layout 里的 "POSITION"
    float2 TextureCoordinate : TEXCOORD; // ← 对应 Input Layout 里的 "TEXCOORD"
};

struct VS_OUTPUT {
    float4 Position : SV_Position;       // SV_ 前缀 = 系统语义，有特殊含义
    float2 TextureCoordinate : TEXCOORD; // 自定义语义，从 VS 传到 PS
};
```

详见 [[D3D11.InputLayout]]

---

### Vertex Shader

每个顶点执行一次：

```hlsl
VS_OUTPUT vertex_shader(VS_INPUT IN)
{
    VS_OUTPUT OUT = (VS_OUTPUT)0;
    OUT.Position = mul(IN.ObjectPosition, WorldViewProjection); // 坐标变换
    OUT.TextureCoordinate = get_corrected_texture_coordinate(IN.TextureCoordinate);
    return OUT;
}
```

详见 [[D3D11.VertexShader]]

---

### Pixel Shader

每个像素执行一次，输出到 Render Target：

```hlsl
float4 pixel_shader(VS_OUTPUT IN) : SV_Target  // SV_Target = 输出到当前绑定的 RT
{
    return ColorTexture.Sample(ColorSampler, IN.TextureCoordinate);
}
```

---

### Technique / Pass

```hlsl
technique10 main10
{
    pass p0                                              // 一个 Pass = 一道工序
    {
        SetVertexShader(CompileShader(vs_4_0, vertex_shader()));
        SetGeometryShader(NULL);
        SetPixelShader(CompileShader(ps_4_0, pixel_shader()));
        SetRasterizerState(DisableCulling);              // 关闭背面剔除
    }
}
```

`pass` 对应渲染管线中的一次完整工序，详见 [[rendering/Deferred-Rendering]]

---

### UV 翻转

```hlsl
#define FLIP_TEXTURE_Y 1

float2 get_corrected_texture_coordinate(float2 uv)
{
#if FLIP_TEXTURE_Y
    return float2(uv.x, 1.0 - uv.y);  // Y 轴翻转
#else
    return uv;
#endif
}
```

原因：DirectX UV 原点在左上角，OpenGL 和大多数 DCC 工具原点在左下角，两者 Y 轴方向相反。
