# Input Layout

An Input Layout defines how the GPU extracts vertex attributes from a vertex buffer and maps them to the Vertex Shader's input struct.

---

## 为什么需要 Input Layout

顶点缓冲上传到 GPU 后只是一堆**原始字节**，GPU 不知道哪些字节是 Position，哪些是 UV。Input Layout 就是这份"切分说明书"。

```
顶点缓冲（原始字节）：
  [12B Position][4B Color][8B UV][12B Normal]...

Input Layout 说：
  前12字节 → POSITION
  接下来4字节 → COLOR
  接下来8字节 → TEXCOORD
  接下来12字节 → NORMAL

VS_INPUT 里：
  float3 Pos   : POSITION   ← 拿到了
  float4 Color : COLOR      ← 拿到了
  float2 UV    : TEXCOORD   ← 拿到了
  float3 Normal: NORMAL     ← 拿到了
```

## D3D11_INPUT_ELEMENT_DESC

```cpp
D3D11_INPUT_ELEMENT_DESC inputElementDesc[] = {
    { "POSITION",  0, DXGI_FORMAT_R32G32B32_FLOAT, 0, 0,                            D3D11_INPUT_PER_VERTEX_DATA, 0 },
    { "COLOR",     0, DXGI_FORMAT_R8G8B8A8_UNORM,  0, D3D11_APPEND_ALIGNED_ELEMENT, D3D11_INPUT_PER_VERTEX_DATA, 0 },
    { "TEXCOORD",  0, DXGI_FORMAT_R32G32_FLOAT,    0, D3D11_APPEND_ALIGNED_ELEMENT, D3D11_INPUT_PER_VERTEX_DATA, 0 },
    { "TANGENT",   0, DXGI_FORMAT_R32G32B32_FLOAT, 0, D3D11_APPEND_ALIGNED_ELEMENT, D3D11_INPUT_PER_VERTEX_DATA, 0 },
    { "BITANGENT", 0, DXGI_FORMAT_R32G32B32_FLOAT, 0, D3D11_APPEND_ALIGNED_ELEMENT, D3D11_INPUT_PER_VERTEX_DATA, 0 },
    { "NORMAL",    0, DXGI_FORMAT_R32G32B32_FLOAT, 0, D3D11_APPEND_ALIGNED_ELEMENT, D3D11_INPUT_PER_VERTEX_DATA, 0 },
};
```

### 每个字段的含义

| 字段 | 说明 |
| :--- | :--- |
| `"POSITION"` | Semantic 名字，对应 HLSL `struct VS_INPUT` 里的 `: POSITION` |
| `0`（第2个）| Semantic Index，有多个同名时区分（如 TEXCOORD0, TEXCOORD1）|
| `DXGI_FORMAT_...` | 数据格式（几个分量，精度多少）|
| `0`（第4个）| 顶点缓冲槽号（InputSlot），通常填 0 |
| `偏移量` | 该属性在一个顶点内的字节偏移 |
| `D3D11_INPUT_PER_VERTEX_DATA` | 每顶点数据（区别于 Instance 数据）|

### D3D11_APPEND_ALIGNED_ELEMENT

代替手动计算偏移，意思是**"紧接上一个属性，自动对齐"**：

```
POSITION:  R32G32B32 = 12字节，偏移 0
COLOR:     R8G8B8A8  =  4字节，偏移 12  ← APPEND 自动算
TEXCOORD:  R32G32    =  8字节，偏移 16  ← APPEND 自动算
TANGENT:   R32G32B32 = 12字节，偏移 24  ← APPEND 自动算
BITANGENT: R32G32B32 = 12字节，偏移 36  ← APPEND 自动算
NORMAL:    R32G32B32 = 12字节，偏移 48  ← APPEND 自动算

一个顶点总共：60 字节
```

## Semantic：连接 Layout 和 VS_INPUT 的桥

连接靠的**不是变量名，而是 Semantic**：

```hlsl
struct VS_INPUT
{
    float3 Position  : POSITION;   // "POSITION" 和 Layout 里的 "POSITION" 对上
    float4 Color     : COLOR;
    float2 TexCoord  : TEXCOORD;
    float3 Tangent   : TANGENT;
    float3 Bitangent : BITANGENT;
    float3 Normal    : NORMAL;
};
```

变量名叫什么无所谓，Semantic 必须匹配：
```hlsl
float3 aaa : POSITION;  // 完全合法，名字不重要
```

## CreateInputLayout

```cpp
UINT numElements = ARRAYSIZE(inputElementDesc);

hr = device->CreateInputLayout(
    inputElementDesc,             // 布局描述数组
    numElements,                  // 元素数量
    vertexShaderByteCode.data(),  // VS 编译后的字节码 ← 用于验证 Semantic 匹配
    vertexShaderByteCode.size(),
    &inputLayout
);
```

> D3D11 会验证 Layout 里的 Semantic 是否与 VS 的 `VS_INPUT` 完全匹配，不匹配则创建失败。

## 绑定与使用

```cpp
// 绑定 Input Layout
context->IASetInputLayout(inputLayout);

// 绑定顶点缓冲
UINT stride = sizeof(Vertex);
UINT offset = 0;
context->IASetVertexBuffers(0, 1, &vertexBuffer, &stride, &offset);

// Draw
context->DrawIndexed(indexCount, 0, 0);
```
