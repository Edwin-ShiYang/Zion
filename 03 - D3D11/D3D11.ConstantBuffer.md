# Constant Buffer

A **Constant Buffer** is a specialized buffer used to pass data from the CPU to shader programs (HLSL). Unlike vertex buffers (每顶点不同), Constant Buffer 里的数据**整个 Draw Call 共享同一份**。

典型用途：WorldViewProjection 矩阵、光源位置、材质参数……

---

## 对接方式：Register 槽号

不像顶点缓冲需要 Input Layout，Constant Buffer 靠**寄存器槽号**对接：

```cpp
// CPU：绑定到 b0 槽
context->VSSetConstantBuffers(
    0,        // ← 槽号 b0
    1,
    &myCBuffer
);
```

```hlsl
// HLSL：声明在 b0 槽
cbuffer CBufferPerObject : register(b0)
{
    float4x4 WorldViewProjection;
    float4x4 World;
    float4   Color;
}
```

槽号对上，数据就到了。无需验证，**程序员自己保证一致**。

---

## 三种 Register 类型

```hlsl
cbuffer   MyBuffer  : register(b0)  // b = constant buffer
Texture2D MyTexture : register(t0)  // t = texture
SamplerState MySampler : register(s0)  // s = sampler
```

---

## 内存布局必须完全匹配

CPU 结构体和 HLSL cbuffer 的字段**顺序、大小必须一致**：

```cpp
// CPU 端
struct CBPerObject
{
    DirectX::XMMATRIX WorldViewProjection;  // 64 字节
    DirectX::XMMATRIX World;                // 64 字节
    DirectX::XMFLOAT4 Color;               // 16 字节
};
```

```hlsl
// HLSL 端（顺序必须相同！）
cbuffer CBufferPerObject : register(b0)
{
    float4x4 WorldViewProjection;  // 64 字节
    float4x4 World;                // 64 字节
    float4   Color;                // 16 字节
}
```

顺序错了数据就乱了，**D3D11 不会报错**——比顶点缓冲更危险。

---

## ⚠️ 16 字节对齐规则

HLSL 的 cbuffer 要求每个变量**不能跨越 16 字节边界**：

```hlsl
// ❌ 危险写法
cbuffer Bad : register(b0)
{
    float  A;      // 偏移 0，  4字节
    float3 B;      // 偏移 4，  12字节 → 偏移4+12=16，刚好但下一个会出问题
    float3 C;      // 偏移 16， 12字节，没问题
    float  D;      // 偏移 28， 4字节，没问题
    float2 E;      // 偏移 32， 8字节
    float3 F;      // 偏移 40， 12字节 → 跨越了 48 这个边界！
}
```

```hlsl
// ✅ 安全写法：大的放前面，或手动补齐
cbuffer Safe : register(b0)
{
    float4x4 Matrix;    // 64字节，天然16字节对齐
    float4   Vec;       // 16字节，天然对齐
    float    Scalar;    // 4字节
    float3   Padding;   // 手动补齐到 16字节
}
```

**最简单的规则：尽量用 `float4` 和 `float4x4`，避免单独的 `float`/`float2`/`float3`。**

## 创建与更新

```cpp
// 创建
D3D11_BUFFER_DESC cbDesc = {};
cbDesc.ByteWidth      = sizeof(CBPerObject);
cbDesc.Usage          = D3D11_USAGE_DYNAMIC;
cbDesc.BindFlags      = D3D11_BIND_CONSTANT_BUFFER;
cbDesc.CPUAccessFlags = D3D11_CPU_ACCESS_WRITE;

device->CreateBuffer(&cbDesc, nullptr, &constantBuffer);

// 每帧更新（Map / Unmap）
D3D11_MAPPED_SUBRESOURCE mapped = {};
context->Map(constantBuffer, 0, D3D11_MAP_WRITE_DISCARD, 0, &mapped);
memcpy(mapped.pData, &cbData, sizeof(CBPerObject));
context->Unmap(constantBuffer, 0);

// 绑定到 VS 的 b0 槽
context->VSSetConstantBuffers(0, 1, &constantBuffer);
```

---

## 常见更新频率分类

| cbuffer 名 | 更新频率 | 典型内容 |
| :--- | :--- | :--- |
| `CBufferPerFrame` | 每帧一次 | 相机矩阵、时间、光源全局信息 |
| `CBufferPerObject` | 每个物体一次 | WorldMatrix、材质参数 |
| `CBufferPerMaterial`| 每种材质一次 | 颜色、粗糙度、金属度 |

按更新频率分开，减少不必要的 Map/Unmap 次数。
